Things to remember

Naming

  • When naming a file add underscores in place of spaces file_name_thing.fileType
  • When naming a class or an ID use Camel case classOrIdName
  • let names describe the fuction of the class or ID

coding style

  • //paragraph

    We tend to add comments around code blocks to indecate where they end and start and what their function is.

    //end of paragraph

indentation

  • Don't forget to properly indent. click HERE to get more instructions of how to properly indent
  • we use tabs with width 4 to indent

Styling

  • an external stylesheet is used
  • all the styling elements should be contained in the external stylesheet, no in-tag styling

Don't forget to...

  • close tags
  • add comments explaining what is the code is meant to do.
